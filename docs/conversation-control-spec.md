# AI Chat Interview Conversation Control

## 深掘り判定ロジック

1. 主質問 Q1〜Q7 は固定で進行する。
2. 深掘り候補は `projects.probe_policy.target_question_codes` と `questions.ai_probe_enabled` の両方で制御する。
3. `projects.probe_policy.blocked_question_codes` に入った質問では深掘りしない。
4. 回答が `short_answer_min_length` 未満、または抽象語パターンに一致した場合だけ深掘り候補にする。
5. `max_probes_per_answer` と `max_probes_per_session` のどちらかに到達したら深掘りを打ち切る。
6. `user_declined` 系の文言なら無理に深掘りせず次へ進む。
7. 回答が `sufficientAnswerMinLength` を超え、抽象回答でもなければ次へ進む。
8. 深掘り文面は `question_config.conversationControl.coreInfoPrompt` を優先し、比較用のコア情報だけを聞く。

## 会話制御ルール一覧

- 主質問の文面と順序は既存の Q1〜Q7 を維持する。
- 追加質問は比較のために不足しているコア情報だけを補う。
- 追加質問は 1 返信 2 文以内を基本にし、80 文字前後で切る。
- 箇条書きはヘルプや設問表示に限定し、通常応答では使わない。
- 深掘り対象は既定で `Q2`, `Q3`, `Q5`, `Q6`。
- 深掘り禁止は既定で `Q1`, `Q4`, `Q7`。
- `start` を途中で再送しても新規セッションは作らず、進行中セッションの続きを返す。
- `最初から` は進行中セッションだけを破棄して Q1 から作り直す。
- 完了済みセッションは `resume` 不可、`最初から` も既定では不可にする。

## 異常入力時の応答一覧

- 空回答: 短くてよいので文字での回答を促し、現在の質問を再表示する。
- 単一選択の不正値: 番号か選択肢名で 1 つだけ送るよう再案内する。
- 複数選択の不正値: 番号か選択肢名をカンマ区切りで送るよう再案内する。
- Yes/No の不正値: `1=はい`, `2=いいえ` を再案内する。
- 尺度回答の不正値: 許容範囲の数字を再案内する。
- スタンプ: 回答として保存できないことを伝え、文字回答を促す。
- 画像: 画像は読めないことを伝え、文字回答を促す。
- その他メッセージ: 対応外形式であることを伝え、文字回答を促す。
- `やめる`: セッションを `abandoned` にして中断案内を返す。
- `最初から`: 進行中なら `abandoned` にして新規セッションを作る。
- `start` 再送: 進行中セッションがあれば現在の質問を返す。
- `unfollow`: セッションを `abandoned`、回答者を `dropped` に更新する。

## 完了フロー仕様

1. 最終質問回答保存後、次質問がなければ `sessions.status=completed` に更新する。
2. 同時に `current_phase=completed`, `current_question_id=null`, `completed_at=now()` を保存する。
3. `messages` に `session_completed` の system log を保存する。
4. 完了後に分析処理を実行する。
5. 完了ポイント、継続ポイント、案件ボーナスを既存ルールどおり付与する。
6. ランク同期後、完了 Flex Message と必要ならランクアップ通知を返す。
7. 完了後の `resume` は不可。既定では再作成も不可にする。

## テスト観点一覧

- Q1〜Q7 が既存順序どおり進むこと。
- Q2/Q3/Q5/Q6 で短文回答時のみ深掘りが入ること。
- Q1/Q4/Q7 では短文でも深掘りしないこと。
- 1 回答あたりの深掘り上限を超えないこと。
- 1 セッションあたりの深掘り上限を超えないこと。
- 深掘り回答が十分なら次の主質問へ進むこと。
- 深掘り回答が短いままで上限未満なら再度だけ確認できること。
- `特にない` などの辞退表現では無理に深掘りしないこと。
- 単一選択、複数選択、yes/no、scale の不正入力再案内が正しいこと。
- 空回答、スタンプ、画像で現在質問を維持したまま再案内できること。
- `やめる` で `abandoned`、`最初から` で新規セッション開始になること。
- 途中で `start` を送っても重複セッションが増えないこと。
- `unfollow` で `dropped` と `abandoned` が保存されること。
- 最終質問後に `completed_at` とポイント付与が一度だけ実行されること。
