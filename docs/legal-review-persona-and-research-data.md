# 書類見直し: 仮想ペルソナ（ai-person連携）／自主調査データの企業提示

> 作成: 2026-07-13 / 対象: 回答者向け利用規約・プライバシーポリシー・企業向け利用規約
> 起点: 二面市場の鶏卵問題の打ち手（①カテゴリ定点調査を営業武器にする ②ペルソナハブ連携）が、
> どちらも「回答者データを本人の回答目的以外に流す」ため、同じ書類に効く。

## 1. すでに許諾されている範囲（＝作り直し不要）

回答者向け利用規約 第10条・プライバシーポリシー 第3条/第5条/第8条が、以下をすでに明示している。

- 回答データ等の**統計化・匿名加工**
- **統計化・匿名加工された情報のクライアント企業への提供**（第三者提供に該当しない扱い）
- クライアント企業の商品開発・マーケティング分析・業務改善のための情報提供
- AIチャットインタビュー機能の**精度向上**
- 統計化・匿名加工された情報は、**退会・削除後も保有・利用できる**

→ **自主調査データを企業に提案する行為は、この範囲でほぼ実施できる。**
→ **ペルソナハブへの供給も同様**。ハブ連携の設計（`ai-person/docs/handoff-ai-chat-interview-integration.md`）が
「生の個人回答・会話生テキストはどの経路でも出さない／セグメント集計値と trait 観測値のみ／k<10 は送らない」を
契約にしているため、**送る中身が最初から『統計化された情報』に限定されている**。設計が守りに入っている分、書類が追いつく。

## 2. 穴（見直しが必要な6点）

### 🔴 穴1: 「公表」の利用目的が無い（最優先・時限性あり）

規約が許すのは「**クライアント企業への提供**」。自主調査レポートは PR・プレスリリース・SEO記事・営業資料として
**不特定多数に公表**するもので、これは書かれていない。

- 直し: 利用規約 第10条1項 / PP 第3条 の利用目的に
  「統計化・匿名加工した調査結果を、レポート・記事・プレスリリース等として**公表**すること」
  「当方サービスの**広告宣伝・営業資料**への利用」を追加。
- **時限性**: 利用目的の追加は、先に直せば以後の同意でカバーされる。
  **営業を始めてから直すと、過去に集めた回答は公表に使えない**（遡って同意は取れない）。
  自主調査の価値は定点＝時系列なので、**最初の1本を撃つ前に直すかどうかで、資産になるかゴミになるかが決まる。**

### 🔴 穴2: verbatim（生の発言引用）の根拠が無い

AI深掘りインタビューの最大の売り物は「なぜやめたか」の**生の言葉**。しかし規約が許すのは
「統計化**又は**匿名加工」だけで、自由記述の生テキストは匿名加工情報の法定加工基準を満たさない限りどちらでもない。

- 直し: 「回答内容の**抜粋・引用**（個人を特定できない範囲で、表現を改変せずに引用する場合を含む）」を明記。
- ここを塞がないと、**一番売れる素材が使えない**。提案書にも自主調査レポートにも効く。
- 併せて運用ルール: 引用前に要配慮個人情報・特異記述のスクリーニングを通す（規約 第8条が
  「要配慮個人情報を自発的に投稿しないよう注意」としているが、混入は起きる前提で設計する）。

### 🟡 穴3: PP 7.2「AI学習への非利用」と仮想ペルソナの見え方

7.2 の主語は OpenAI（「AIサービス提供者は…自社モデルの学習に使用しない」）なので、
**自社ペルソナ生成を禁じてはいない**。だが利用者・取引先からは「AI学習に使わないと書いてあるのにAIペルソナを作っている」と読まれる。

- 直し: 7.2 の主語を「AIサービス提供者のモデル学習」に限定して明確化し、**別項で**
  「当方は回答データを統計化したうえで、セグメント代表プロファイル（合成ペルソナ）の生成に利用することがある」と書き分ける。
- **黙っておくのが一番危ない**（説明できない状態で外部に見つかる）。書けば普通の話。

### 🟡 穴4: CC BY 4.0 帰属表示の射程（盲点）

ペルソナの土台は Nemotron-Personas-Japan（CC BY 4.0）。ai-person アプリのフッターには帰属表示済み。
しかし **CC BY は「使った媒体」に表示義務が及ぶ**。ペルソナを企業向け提案書・調査レポート・Hibi の画面に載せた瞬間、
**その媒体にも帰属表示が必要**になる。ai-person 内だけの話にしていると漏れる。

- 直し: 提案書テンプレ・レポート雛形・企業向け画面のフッターに帰属表示を定型で埋め込む。

### 🟡 穴5: 合成明示と「実測／予測」のラベル分離が対企業側に無い

ai-person 側は F-3（合成明示）・E-2（合成は一次スクリーニング、最終判断は実在ユーザー）を決定済みで、
ハブDBも `survey_traits`（実測）と `estimated_traits`（推定）を分離している。
**この分離を納品物・提案書・企業向け画面で崩すと、事実上のデータ捏造になる。**

- 直し: 企業向け利用規約（…0005）に「合成ペルソナの出力は実在人物の回答ではなく参考情報である」旨の免責を追加。
- 直し: 提案書・レポートで **実測値と予測値のラベル分離を必須**にする（ハブのスキーマ分離をそのまま外まで持ち出す）。

### 🟡 穴6: 規約が「クローズドβ版」のまま

利用規約 第1条・第2条が「クローズドβ版」「招待した方に限り提供」「正式リリース時に改めて見直す」。
**企業に有償で売る＝もうβではない**。企業向け利用規約で商用取引を始めるのに、回答者向け規約がβのままだと、
AS-IS無保証（第3条）と有償役務提供の整合が崩れる。

### 🟠 穴7（実装確認事項）: 店舗経由（非会員）回答者への規約提示

店舗QR流入は会員登録せず回答できる経路がある（B-first・entry_code）。その回答を自主調査やペルソナに使うなら同意が要る。
`survey.ejs` は `consent-check` を呼んでいるが、**非会員のまま完了する経路で規約が提示されているか**は要確認。

## 3. 実行順序

1. **【営業開始前・1回で出す】回答者向け規約＋PP の v2.0**
   β解除（穴6）＋公表目的（穴1）＋引用/verbatim（穴2）＋ペルソナ生成の明示（穴3）を**まとめて1版**にする。
   **再同意は回数を増やさないのが鉄則**なので、小出しにしない。既存の版管理・再同意基盤（`document_versions` +
   `consentService.publishNewVersion`）に載る。
2. **ハブ経路の制約を「守りの資産」として文書化**
   k≥10・生回答非送出・クリーニング裁定後のみ、を企業向けにも明言する（「うちは生回答を外に出しません」は営業文句になる）。
3. **企業向け成果物のルール**: 合成/実測のラベル分離（穴5）＋合成免責（穴5）＋CC BY 帰属（穴4）を
   提案書テンプレと企業向け規約に入れる。
4. **店舗（非会員）経路の規約提示を実装確認**（穴7）。

## 4. 未決（本人判断が要る）

- **合成ペルソナの回答（予測分布）を企業提案に出すか、社内の一次スクリーニングに留めるか。**
  ai-person 側の決定表 G-4 は「⏸ 仮置き: まず社内のみ。後から外に出せる設計」[要確認] のまま。
  今回の営業構想は**これを外に出す話**なので、この仮置きを昇格させる必要がある。
  外に出すなら穴4・穴5 が即時必須。社内限定なら後回しにできる。
