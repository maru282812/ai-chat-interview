# 規約 v2.0 条文案（公表・引用・合成ペルソナの明示）

> 作成: 2026-07-13 / 前提: [legal-review-persona-and-research-data.md](legal-review-persona-and-research-data.md)
> 方針: **後から「使えない」を絶対に起こさないため、この段階で使途を明示する。**
> ⚠️ 本書は条文の**案**であり、法的助言ではない。発行前に専門家（弁護士）の確認を通すこと。

## 0. 設計方針 — 「広く、しかし具体的に」

個人情報保護法は利用目的を「**できる限り特定**」せよと求める（17条）。
「あらゆる目的に自由に使用できる」式の**白紙委任は、かえって効力を争われる**。
したがって v2.0 は **包括的な一文ではなく、具体的な用途を列挙して広く取る**。

同時に、**やらないことを書く**（生回答は外に出さない・k<10 は出さない・要配慮情報は引用しない）。
これは実装がすでにそうなっている（ハブ連携の k≥10 契約・生回答非送出）ので**書いても何も失わず、信頼と営業文句が手に入る**。

## 1. 遡及適用がこの改訂の肝

**過去に集めた回答は、旧規約の利用目的の範囲でしか使えない。** ただし本人の再同意があれば、過去分にも新目的を及ぼせる。
そのため v2.0 には**遡及条項を必ず入れる**（下記 第10条 第5項）。これが入っていないと、
再同意しても「v2.0 同意日より前の回答」が公表・ペルソナに使えず、**定点調査の過去分＝一番価値のある部分が死ぬ**。

**再同意しなかった利用者の回答は、新目的では使えない。** → 実装タスク（§6）で除外フィルタが必要。

## 2. 回答者向け利用規約（d0000000-…0001）v2.0

### 第1条・第2条・第3条: クローズドβ表記の解除

- 第1条1項の「【現在クローズドβ版として提供中です】」ブロックを削除。
- 第2条（クローズドβ版としての提供）を削除し、以降の条番号を繰り上げ（または第2条を「本サービスの提供」に差し替え）。
- 第3条（現状有姿・無保証）は**維持**する。回答者に対しては無償役務であり、AS-IS は合理的。
  企業に対する有償役務の保証水準は企業向け規約で別途定めるため、矛盾しない。

### 第10条（回答データ等の取扱い）— 第1項に追加する号

現行の列挙（サービス提供・AI精度向上・統計化/匿名加工・クライアント企業への提供・商品開発/マーケ分析）に、以下を**追加**する。

```
(f) 統計化又は匿名加工された情報を、調査レポート、記事、プレスリリース、当方の広告宣伝物、
    営業資料その他の媒体により公表・公開すること（当方が自ら企画・実施する調査の結果を
    公表する場合を含みます）。

(g) 回答内容の抜粋・引用（個人を特定できない範囲において、回答の表現を改変せずに引用する
    場合を含みます）を、前号の公表物及びクライアント企業への提供物に掲載すること。

(h) 回答データ等を統計化した情報に基づき、性別・年代・地域その他の属性区分ごとの代表的な
    人物像（以下「合成ペルソナ」といいます）を生成し、当方及びクライアント企業のマーケティング
    分析・商品開発の用に供すること。合成ペルソナは実在の個人ではなく、統計値から合成された
    ものです。

(i) 合成ペルソナによる回答の予測を生成すること、及び当該予測の正確性を実際の回答分布と
    突き合わせて検証・改善すること。

(j) 当方が運営する関連サービス及び関連システムにおいて、前各号の目的で回答データ等を
    利用すること。
```

### 第10条 — 新設する第4項（当方の遵守事項＝やらないことの明示）

```
4. 当方は、前項各号の利用にあたり、以下を遵守します。

 (1) 個々のユーザーの回答原文及びAIチャットインタビューの対話履歴そのものを、クライアント企業
     その他の第三者に提供しません。第三者に提供するのは、統計化又は匿名加工された情報、及び
     個人を特定できない範囲での抜粋・引用に限られます。

 (2) 属性区分ごとの集計値を第三者に提供し又は公表する場合、当該区分に属するユーザーが
     10名未満となるものは、提供・公表しません。

 (3) 個人情報の保護に関する法律第2条第3項に定める要配慮個人情報に該当し得る記述は、
     前項(g)の抜粋・引用の対象から除外します。

 (4) 合成ペルソナには、個々のユーザーの回答原文及び対話履歴を転写しません。
```

> この4項は**実装がすでにそうなっている**（ハブ連携は k≥10 のセグメント集計値と trait 観測値のみを送り、
> 生の個人回答・会話テキストはどの経路でも外に出さない）。書いても失うものが無く、
> 「うちは生回答を外に出しません」という**営業文句**にもなる。

### 第10条 — 新設する第5項（遡及適用）

```
5. 本条の規定は、ユーザーが本規約に同意した時点より前に当方が取得した回答データ等についても
   適用されます。
```

また、当方は、統計化又は匿名加工された情報及び個人を特定できない範囲での回答の抜粋・引用を、

### 第7条（AI処理の透明性）— 7.2 の主語を限定し、7.4 を新設

**7.2 の改訂**（現行は「AI学習への非利用」という見出しのため、「AIに一切使わない」と読まれる）:

```
### 7.2 AIサービス提供者によるモデル学習への非利用

AIサービス提供者（OpenAI OpCo, LLC）は、API経由で送信されるデータを同社のモデルの学習に
使用しないことを契約又は利用規約において明示しています（2026年6月時点）。当方は、
AIサービス提供者がデータを学習目的に使用しないことを定期的に確認します。

本項は、AIサービス提供者による学習利用について定めるものであり、当方自身による回答データの
分析・統計化・合成ペルソナ生成（第7.4項）を制限するものではありません。
```

**7.4 の新設**:

```
### 7.4 合成ペルソナの生成について

当方は、回答データを統計化した情報に基づき、属性区分ごとの代表的な人物像（合成ペルソナ）を
生成し、マーケティング分析等に利用することがあります。

- 合成ペルソナは実在の個人ではなく、統計値から合成されたものです。
- 個々のユーザーの回答原文・対話履歴が合成ペルソナに転写されることはありません。
- 合成ペルソナの生成には、属性区分ごとに10名以上のユーザーの回答から得られた集計値のみを
  用います。
```

> **黙っているのが一番危ない**箇所。「AI学習に使わないと書いてあるのにAIペルソナを作っている」と
> 外部に指摘される前に、自分から書き分けておく。書けば普通の話で終わる。

## 4. LINE登録時同意文（63000000-…0003）

登録時に実際に提示される要約文。**本体（規約・PP）と食い違うと同意の有効性が揺らぐ**ので、必ず同時改訂する。

追加する要約行の案:

```
・ご回答は、統計データや匿名化した情報として、調査レポートの公表や企業への提供に利用します
・個人が特定されない範囲で、ご回答の一部を引用することがあります
・ご回答を統計化した情報から、属性ごとの「代表的な人物像」を作成することがあります
  （実在の個人ではありません）
・個々のご回答の原文や会話の履歴を、そのまま企業に渡すことはありません
```

### CC BY 4.0 帰属表示（定型文）

**ペルソナを載せる媒体すべて**（提案書・調査レポート・企業向け画面）のフッターに入れる。
ai-person アプリのフッターにあるだけでは足りない（**CC BY は「使った媒体」に表示義務が及ぶ**）。

```

Persona data derived from NVIDIA Nemotron-Personas-Japan, licensed under CC BY 4.0.
```

## 6. 発行手順と実装タスク

### 発行（追加実装は不要）

`consentService.getPendingGlobalConsents` は「同意済み version_id ≠ 現行 current_version_id」で
pending を返す（[consentService.ts:46](../src/services/consentService.ts#L46)）。
→ **v2.0 を publish した瞬間、既存ユーザー全員が自動的に再同意対象になる。**
既存の `/liff/consent-check` ゲート（survey / mypage / project-detail から呼ばれている）がそのまま働く。

**4書類を同時に v2.0 として publish する**（規約・PP・LINE登録時同意文・企業向け規約）。
再同意は回数を増やさないのが鉄則なので、小出しにしない。

### 実装タスク（新規）

1. **v2.0 未同意者の除外フィルタ**
   再同意しなかった利用者の回答は、新目的（公表・引用・ペルソナ供給）では**使えない**。
   以下の出力経路で「回答者が v2.0 に同意済みか」を判定して除外する必要がある:
   - 公表用の自主調査集計
   - ハブへの actuals / trait-observations push
   - 提案書用の verbatim 抽出
   `user_consent_records` に document_version_id があるので判定は可能。**集計側にこのフィルタが無いと、
   規約を直した意味が無くなる**。

2. **要配慮個人情報スクリーニング**（第10条4項(3)の履行）
   verbatim 引用の前に、要配慮情報・特異記述を弾く工程を入れる。規約で「除外する」と約束する以上、
   運用が要る。

3. **実測／予測のラベル分離**を成果物側で担保
   ハブは `survey_traits`（実測）と `estimated_traits`（推定）を既に分離している。
   **この分離を提案書・レポート・画面まで持ち出す**。混ぜると事実上のデータ捏造になる。

4. **店舗経由（非会員）回答者への規約提示**の実装確認
   会員登録せず回答できる経路（entry_code / B-first）で規約が提示されているか。
   提示されていない回答は、新目的で使えない。

## 7. やらないこと（意図的に広げない）

- **回答原文・対話履歴そのものの第三者提供**: 書かない。実装もそうなっていない。ここを開けると
  「生回答を出さない」という守りの資産と営業文句を同時に失う。
- **個人単位のペルソナ複製**（ai-person の注入方式(c) genagents 式）: 対象外。
  ai-person 側でも「(c)個人複製は対象外」と決定済み。将来やるなら本人同意・削除権の別建てが要る。
- **包括的な白紙委任条項**: §0 のとおり、逆効果。
