# 統計エクスポート改修 実装状況（2026-06-29）

改修指示「アンケート・インタビュー結果を外部統計ソフトで扱えるようにする」（全23項目）の実装状況。
方針: **後方互換（§12）を守り、既存CSV出力・回答フローは一切変更しない追加実装**。今回はDB変更なしで
完結する「エクスポート層」と「送付前バリデーション層」を純関数中心で実装。DB＋ランタイム改修が必要な
項目は次フェーズへ整理した。

## 実装済み（このブランチ: feat/statistical-export-engine）

### コードブック導出層 — [src/lib/codebook.ts](../src/lib/codebook.ts)（§7 / §20）
- 設問→統計変数定義（variable_name / data_type / measure_type / allowed_values /
  multi_select_encoding / free_text_policy / aggregation_policy / scale範囲）をマイグレーション無しで導出。
- 既定値は question_type / question_config から導出。管理者上書きは `question_config.meta.cleaning` から読む。
- 欠損値センチネル `DEFAULT_MISSING_CODES`（無回答=`.` / 不明=99 / 拒否=98 / 対象外=97 /
  スクリーニング落ち=96 / 分岐非表示=95 / 未到達=94）。変数名は統計ソフト向けに安全化（`__free_comment__`→`free_comment`）。

### エクスポート生成エンジン — [src/lib/statExport.ts](../src/lib/statExport.ts)（§2 §8–§11 §15 §19 §20 §21）
- `buildWideRows` 1回答者1行・**列は送付時マスター順（既定 sort_order, §12）**・多重選択は **one-hot 展開（§15）**
  `<var>_<option_code>=0/1` ＋ `<var>_other_text`・欠損はセンチネル・テスト回答は既定除外（§17）。
- AI深掘り統合値（§9）: ai_probe_enabled 設問に `_final_answer_text / _cleaned_value / _extracted_json /
  _probe_triggered / _probe_count / _probe_reason / _probe_questions_json / _probe_answers_json`。
  **比較単位は常に元の question_code（§8）。深掘りは回答者数に加算しない。**
- `buildLongRows` 1回答1行・primary / ai_probe を別行・`parent_answer_id` 紐づけ・実表示順(display_position)・
  §10 深掘りバイアス用メタ（probe_reason / probe_question_text / probe_model / probe_prompt_version /
  probe_completion_score / probe_confidence）。
- `buildCodebookRows` / `buildSnapshotJson`（読み取り側スナップショット）/ `buildRandomizationLogRows`。
- `toCsvRfc4180` UTF-8 BOM + RFC4180 + CRLF（§21）。
- §19: 出力主キーは擬似匿名 `respondent_key`（= respondent.id）。line_user_id 等は出力しない。
